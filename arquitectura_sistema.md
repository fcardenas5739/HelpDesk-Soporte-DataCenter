# Arquitectura lógica del sistema Help Desk

El sistema se organiza en cuatro capas principales:

1. Capa de presentación: permite al usuario registrar y consultar tickets.
2. Capa de control: recibe las solicitudes y coordina las acciones del sistema.
3. Capa de modelo: contiene las clases Ticket, Usuario, Técnico, Administrador e Incidente.
4. Capa de persistencia: gestiona el almacenamiento de la información.

La arquitectura propuesta facilita la mantenibilidad, escalabilidad y trazabilidad del sistema.
